## Working Hypotheses

### 1. Surface drift / split-brain stack is real

The docs say Hermes / Commander / Apollo should be driven through:

- `:8642`
- `:8643`
- `:8646`
- `:9119`

But the live machine today is actually operating on:

- `:3000`
- `:8092`
- `:11434`

That mismatch alone can explain operator confusion, session weirdness, and duplicated traffic paths.

### 2. Context bloat/session reuse is still a first-class bug

Prior evidence already traced the browser chat issue to `claude-last-session` reuse. Nothing in this pass contradicts that. Since the browser cockpit `:9119` is down right now, the bug is not currently being re-tested live, but the architecture remains compatible with session state confusion.

### 3. Telegram/OpenRouter bursts are probably control-plane behavior, not hallucinated

The logs confirm real cron-driven outbound activity to Telegram and real OpenRouter usage. The specific “80k tokens every second for 3 seconds” event is not yet directly reproduced from current logs, but the stack absolutely contains:

- scheduled jobs
- Telegram live delivery
- OpenRouter-backed agents
- retries / rate-limit behavior

So a burst caused by automation, duplicate gateway copies, or repeated resume behavior remains plausible.

### 4. GPU driver failure is not the leading explanation for the whole stack

This pass again shows:

- CUDA detects the Blackwell GPU
- the worker lane starts and serves
- model load succeeds inside the Docker `llama.cpp` path

That does not rule out another lane using the GPU poorly, but it does argue against “the card/driver is fundamentally broken.”
